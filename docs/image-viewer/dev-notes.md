# image-viewer 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| 画像ロードを Pillow → QImageReader に変更 | Qt ネイティブの方が JPEG/PNG デコードが高速。EXIF 回転も自動対応。Pillow 依存を削除できる |
| ワーカーで QImage を返し QPixmap 変換はメインスレッドで行う | Qt の規則: QPixmap はメインスレッドのみで生成可能 |
| キャッシュに世代番号を導入 | フォルダ切替後に古いワーカーの結果がキャッシュを汚染するのを防ぐ |
| フィット・ズームをそれぞれ別キャッシュで管理 | ドラッグ中の毎フレームスケーリングを回避し描画を高速化 |
| hasChildren を非同期チェックに変更 | 同期 iterdir() は多フォルダ表示時にメインスレッドをブロックする |

## 発生した問題と対処

| 問題 | 対処 |
|------|------|
| build.bat の ^ 行継続が Windows で動作しない | LF 改行が原因。コマンドを1行に統合 |
| ビルド時 PermissionError（exe が起動中）| ビルド前に taskkill で既存プロセスを終了 |
| PyInstaller バンドル時にアイコンが表示されない | `sys._MEIPASS` を使ったパス解決に変更 |
| タブのトグルボタンが押しても文字が変わらない | `setSizes()` は `splitterMoved` を発火しないため、`_update_toggle_btn` を明示的に呼ぶよう修正 |

## 設計からの変更点

| 変更内容 | 理由 |
|----------|------|
| スワップモードのデフォルトをホイール=移動に変更 | ユーザーの要望 |
| フォルダビューの矢印を非同期チェックに変更 | 初期表示のパフォーマンス改善のため設計変更 |

## 今後の課題

- GIF アニメーションは先頭フレームのみ（動画対応は未実装）
- ネットワークドライブの大量フォルダでの hasChildren パフォーマンス未検証
- お気に入り D&D のネスト深い操作で QStandardItemModel の挙動が不安定になる可能性
