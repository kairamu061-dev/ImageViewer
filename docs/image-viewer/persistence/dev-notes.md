# persistence 開発メモ

## 実装上の判断

| 判断内容 | 理由 |
|----------|------|
| `restore_state` でスワップモードを最初に設定 | `add_new_tab()` 内で `set_swap_mode(_default_swap_mode)` を呼ぶため、タブ生成前に設定が必要 |
| 保存/読み込みエラーをサイレントに握りつぶす | 状態ファイルの問題でアプリが起動できなくなるのを防ぐ |
| `_get_swap_mode()` で既存タブから読む | 保存時点での実際の設定値を取得するため |
